DSA Practice Repository

Welcome to my DSA (Data Structures and Algorithms) practice repository. This repository contains solutions to a variety of DSA problems, categorized by topics. The solutions include problems from popular platforms like LeetCode and follow structured approaches such as Striver's DSA sheet.
📁 Repository Structure

The repository is organized into the following folders:

    Arrays: Problems related to arrays, including searching, sorting, and manipulation techniques.
    Bit Manipulation: Concepts involving bitwise operations and bit-level tricks.
    Bitmask: Solutions using bitmasking approaches for subset generation, dynamic programming, etc.
    Dynamic Programming (DP): Problems involving optimization using memoization, tabulation, and recursive approaches.
    Graphs: Graph traversal techniques, shortest path algorithms, and network flow problems.
    Greedy Approaches: Problems that can be solved using greedy algorithms, focusing on local optimal choices.
    Heaps: Solutions related to heap data structures, including priority queues and heap-based problems.
    LeetCode Daily: Daily LeetCode problems to keep up with regular practice.
    Linked Lists: Concepts involving singly, doubly, and circular linked lists.
    Queues: Problems that utilize queue data structures, including deque and circular queues.
    Recursion: Recursive problem-solving, including backtracking.
    Searching: Search algorithms like binary search, linear search, and their variations.
    Sliding Window: Techniques for solving problems using the sliding window approach.
    Sorting: Different sorting algorithms, including quicksort, mergesort, and insertion sort.
    Stacks: Problems that use stack data structures, including balanced parentheses, postfix expressions, etc.
    Strings: String manipulation, pattern matching, and more.
    Trees: Solutions involving tree data structures, including binary trees, BSTs, and trie-based problems.

📝 Approach

For each problem, the solution is documented with:

    Problem Statement: A brief description of the problem.
    Approach: Explanation of the thought process and strategy used to solve the problem.
    Complexity Analysis: Time and space complexity analysis.
    Code: Well-documented code to aid understanding.
    Edge Cases: Consideration of edge cases and how they were handled.
